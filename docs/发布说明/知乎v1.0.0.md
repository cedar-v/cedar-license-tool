# 你的软件值多少钱？聊聊软件资产化与授权管理

做软件的人都知道，写代码只是第一步。**把代码变成持续盈利的产品**，才是真正有价值的事。

但现实往往是这样：辛辛苦苦开发的产品，别人随便拷贝一份就能用；卖给客户的软件，根本不知道对方复制了多少份；想按年收费，结果客户一次性买断后就没了下文……

这些问题的根源在于：**缺乏一套有效的软件资产保护机制。**

---

## 软件授权是软件资产化的基石

软件授权（License），本质上是把软件的使用权变成一种可量化、可追踪、可交易的数字资产。它解决三个核心问题：

**1. 确权 — 我的软件我做主**

没有授权体系，软件就是「裸奔」状态，谁都能用，谁用了你也不知道。许可证让每一份软件的使用权都有明确的归属和边界。

**2. 变现 — 让软件真正产生收入**

按功能模块授权、按时间订阅、按设备数量计费……授权体系决定了你的商业模式。一次性卖断还是持续收费，授权设计是关键。

**3. 管控 — 知道软件用在了哪里**

通过机器指纹绑定、设备数量限制、到期提醒等机制，你可以清晰掌握软件的使用情况，及时发现异常。

---

## 没有授权管理的代价

很多中小团队和独立开发者一开始不在意授权管理，觉得「先活下去再说」。但等到问题暴露时，往往已经付出了更高代价：

- 客户用免费试用的名义长期白嫖产品
- 软件被无限复制分发，收入流失严重
- 无法基于真实使用数据做产品决策
- 后续想转型订阅制，发现历史客户关系已经无法管理

**软件授权不是大公司的专利。越早建立，越早受益。**

---

## 今天要推荐的工具：CedarLicenseTool

一个痛点：**给客户生成一张许可证，要登录后台、填一堆信息、导出文件、再发给客户**——整个流程走下来，可能要好几分钟。客户催得急的时候，这种效率简直是噩梦。

**CedarLicenseTool** 就是来解决这个问题的：**不用登录管理后台，在本地一键生成并导出客户许可证**。开源免费，开箱即用。

---

## 它能做什么？

**一句话：填写客户名称 + 机器指纹 → 生成许可证文件 → 下载交付。** 整个流程不到 1 分钟。

具体功能：

**密钥管理**
- 一键生成 RSA 密钥对，自动保存到本地
- 启动时自动加载已有密钥，不用每次重复生成
- 支持复制私钥/公钥到剪贴板

**许可证生成**
- 输入客户名称和机器指纹即可生成许可证
- 授权时长灵活：快速选 1/3/6 个月或 1/2/3 年，也支持自定义天数
- 支持「有期限」和「无期限（永久）」两种授权模式
- 支持 JSON 格式扩展功能配置

**导出与分发**
- 许可证以 Base64 编码展示，可直接复制粘贴
- 下载为 `.lic` 文件到本地
- 一键导出存档包（包含公钥 + 许可证文件），方便打包发给客户

**加密安全**
- 采用 RSA-PSS + SHA-256 签名算法，与 [Cedar-v LicenseManager](https://github.com/cedar-v) 完全兼容

---

## 为什么做这个？

我们团队内部在用 Cedar-v 的 LicenseManager 管理授权，以往给客户生成许可证需要登录管理后台，操作路径比较长。对于客户量不大但又需要频繁交付授权的场景，这个流程显得过于繁琐。

所以干脆写了这个工具，把授权流程简化到极致——**本地操作，无需联网后台，生成即可直接交付**。

## 使用方法

### 1. 下载安装

前往 GitHub 下载对应系统的可执行文件，双击运行即可。

> **下载地址**: https://github.com/cedar-v/cedar-license-tool

### 2. 生成密钥对

首次使用，工具会自动生成 RSA 密钥对并保存到本地。之后启动会自动加载，无需重复操作。

### 3. 生成许可证

在工具界面中填写以下信息：
- **客户名称**：你的客户名称
- **机器指纹**：客户设备的唯一标识
- **授权时长**：快速选择 1/3/6 个月或 1/2/3 年，也可以自定义天数
- **授权模式**：有期限或无期限（永久）

填写完成后，点击生成，工具会自动生成许可证。

### 4. 导出交付

- 直接复制 Base64 编码的许可证内容
- 或下载为 `.lic` 文件
- 一键导出公钥 + 许可证存档包，打包发给客户即可
