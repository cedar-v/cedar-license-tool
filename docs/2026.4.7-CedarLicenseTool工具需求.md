# CedarLicenseTool 许可证分发工具需求文档

## 1. 概述

### 1.1 项目背景

开发一款独立的桌面许可证分发工具，方便销售和技术支持人员为客户生成授权许可证文件，无需登录管理后台即可完成许可证的生成与导出。

### 1.2 核心目标

- 生成 RSA 密钥对（私钥/公钥）
- 填写客户信息后生成带有数字签名的许可证文件
- 导出的许可证文件与 license-manager-enterprise 管理系统 **完全兼容**

### 1.3 技术选型

| 组件 | 技术 | 说明 |
|------|------|------|
| 桌面框架 | Wails v2 | Go 后端 + Web 前端的跨平台桌面应用 |
| 后端加密 | Go 标准库 `crypto` | RSA 密钥对生成、许可证签名 |
| 前端框架 | React | UI 渲染 |
| UI 组件库 | shadcn/ui + Tailwind CSS | 还原 Figma 设计 |

## 3. 功能需求

### 3.1 密钥管理

#### 3.1.1 创建密钥对

- **触发方式**：点击「创建密钥对」按钮
- **密钥规格**：
  - 算法：RSA
  - 密钥长度：**2048 位**
  - 私钥格式：**PKCS1**（`-----BEGIN RSA PRIVATE KEY-----`）
  - 公钥格式：**PKIX**（`-----BEGIN PUBLIC KEY-----`）
- **存储方式**：本地文件
  - 私钥文件：`keys/license_private.pem`（权限 0600）
  - 公钥文件：`keys/license_public.pem`（权限 0644）
- **显示**：生成后自动填入界面私钥/公钥区域

> ⚠️ **兼容性要求**：必须与 license-manager-enterprise 后端 `pkg/utils/crypto.go` 保持一致。

#### 3.1.2 密钥加载

- 启动时自动检查 `keys/` 目录是否存在密钥文件
- 存在时自动加载到界面显示区域
- 不存在时显示提示文本

### 3.2 许可证生成

#### 3.2.1 输入字段

| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 客户名称 | 文本 | 是 | 用于备注/归档 |
| 机器指纹 | 文本 | 是 | 设备唯一标识 |
| 授权时长类型 | 单选 | 是 | 有期限 / 无期限 |
| 授权开始时间 | 日期时间 | 否 | 默认为当前时间 |
| 授权结束时间 | 日期时间 | 否 | 根据时长自动计算 |
| 包含授权码信息 | 复选框 | 否 | 是否在许可证中包含授权码详情 |

#### 3.2.2 快速选择时长

预设选项：`1个月`、`3个月`、`6个月`、`1年`、`2年`、`3年`

选择后自动填充自定义时长输入框。

#### 3.2.3 自定义时长

- 输入数值 + 单位选择（月 / 年）
- 最大支持 99 年

### 3.3 许可证文件结构

生成的 `.lic` 文件为 **Base64 编码**的 JSON 字符串，解码后结构如下：

```json
{
  "data": "<原始 JSON 数据（字符串形式）>",
  "signature": "<Base64 编码的 RSA-PSS-SHA256 签名>",
  "algorithm": "RSA-PSS-SHA256"
}
```

#### 原始数据字段（`data` 内容）

```json
{
  "license_key": "LIC-DEVICE-XXXXXXXXXXXX",
  "customer_name": "客户名称",
  "hardware_fingerprint": "设备指纹",
  "start_date": "2026-01-01T00:00:00Z",
  "end_date": "2027-01-01T00:00:00Z",
  "created_at": "2026-04-07T10:00:00Z"
}
```

### 3.4 导出功能

| 功能 | 描述 |
|------|------|
| 下载存档 | 导出包含私钥、公钥、许可证文件的压缩包 |
| 复制许可证 | 将许可证 Base64 字符串复制到剪贴板 |
| 下载许可证 | 导出单个 `.lic` 文件 |

## 4. 加密实现规范

### 4.1 签名算法

| 参数 | 值 |
|------|-----|
| 算法 | RSA-PSS |
| 哈希函数 | SHA-256 |
| Salt 长度 | `PSSSaltLengthEqualsHash` |
| 密钥长度 | 2048 位 |

### 4.2 代码实现要点

密钥生成：

```go
// 参考：license-manager-enterprise/backend/pkg/utils/crypto.go
privateKey, err := rsa.GenerateKey(rand.Reader, 2048)

// 私钥 PEM 编码
pem.EncodeToMemory(&pem.Block{
    Type:  "RSA PRIVATE KEY",
    Bytes: x509.MarshalPKCS1PrivateKey(privateKey),
})

// 公钥 PEM 编码
x509.MarshalPKIXPublicKey(&privateKey.PublicKey)
```

签名：

```go
hashed := sha256.Sum256(data)
signature, err := rsa.SignPSS(rand.Reader, privateKey, crypto.SHA256, hashed[:], &rsa.PSSOptions{
    SaltLength: rsa.PSSSaltLengthEqualsHash,
    Hash:       crypto.SHA256,
})
signatureBase64 := base64.StdEncoding.EncodeToString(signature)
```

### 4.3 与 license-manager-enterprise 的兼容性

生成的许可证文件必须满足以下兼容性要求：

| 检查项 | 要求 |
|--------|------|
| 签名算法 | RSA-PSS with SHA-256 |
| 私钥格式 | PKCS1 (`-----BEGIN RSA PRIVATE KEY-----`) |
| 公钥格式 | PKIX (`-----BEGIN PUBLIC KEY-----`) |
| 签名输出 | Base64 编码 |
| 文件结构 | `{data, signature, algorithm}` |
| 算法标识 | `"RSA-PSS-SHA256"` |

## 5. 数据结构

### 5.1 许可证数据模型

```typescript
interface LicenseData {
  license_key: string;           // 许可证唯一标识，格式：LIC-DEVICE-{HEX}
  customer_name: string;         // 客户名称
  hardware_fingerprint: string;  // 机器指纹
  start_date: string;           // RFC3339 格式开始时间
  end_date: string;             // RFC3339 格式结束时间（无期限时为空）
  created_at: string;          // RFC3339 格式创建时间
}
```

### 5.2 许可证文件结构

```typescript
interface LicenseFile {
  data: string;        // Base64 编码的原始 JSON 字符串
  signature: string;   // Base64 编码的 RSA-PSS-SHA256 签名
  algorithm: string;   // "RSA-PSS-SHA256"
}
```

### 5.3 密钥对结构

```typescript
interface KeyPair {
  privateKey: string;  // PEM 格式私钥
  publicKey: string;   // PEM 格式公钥
}
```

## 6. 用户操作流程

### 6.1 首次使用流程

```
1. 启动应用程序
   ↓
2. 首次使用提示：「请先创建密钥对」
   ↓
3. 点击「创建密钥对」按钮
   ↓
4. 密钥文件保存到 keys/ 目录
   ↓
5. 私钥/公钥自动填入界面显示区
   ↓
6. 填写客户名称、机器指纹
   ↓
7. 选择授权时长
   ↓
8. 点击「生成许可证」
   ↓
9. 预览区显示许可证内容（JSON 格式）
   ↓
10. 复制或下载许可证文件
```

### 6.2 再次使用流程

```
1. 启动应用程序
   ↓
2. 自动加载已有密钥对
   ↓
3. 填写/修改客户信息
   ↓
4. 生成新许可证
```

## 8. 错误处理

| 场景 | 处理方式 |
|------|----------|
| 密钥文件不存在 | 提示用户创建密钥对 |
| 密钥文件损坏 | 提示重新生成密钥对 |
| 许可证生成失败 | 显示错误信息，允许重试 |
| 文件写入失败 | 提示检查目录权限 |
| 输入参数无效 | 表单级别实时校验 |

## 9. 非功能性需求

### 9.1 性能

- 密钥生成时间 < 1 秒
- 许可证生成时间 < 500ms

### 9.2 安全性

- 私钥文件权限：仅所有者可读写（0600）
- 不在日志中记录敏感信息
- 私钥不随许可证文件导出（仅导出公钥）

### 9.3 跨平台

- Windows 10/11
- macOS 11+
- Linux（Ubuntu 20.04+）
